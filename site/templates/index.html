How to use this tool

  1. Go to your Immich instance here: https://my.immich.app/user-settings?isOpen=api-keys
  2. Generate a new API key
  3. Name it to something that you can recognise later on (`immich-tiktok-remover` for example)
  4. You only need the `asset.read` permission for this site to work.
  5. Click create and copy the api key into the `Immich API Key` box above
  6. Paste in your immich server url (including http/https)
  7. Select if you want this took to run every 24hrs

You should notice the tool running by chcecking your trash folder and seeing TikTok videos in there. Please check the trash folder occasionally to check that there haven't been any legitimate files that have been trashed. I haven't had a single issue with this tool since I started using it a year ago but mistakes happen and it's always worth checking

If you want to stop this tool from running, simply delete the API key you created in step #1; the pod on the cluster should stop automatically.

Please note that this tool has a rate limit of 3hrs.

Is it safe?

All the code is available on GitHub here: Immich Tiktok Remover which you're free to check.

There's always a risk when you send your API key to an external service. Any keys sent are only stored in a kubernetes cluster for the deployments to use. To reduce the chance of a leak causing issues with your immich server I would recommend to only add the needed permissions for this tool to work (`asset.read` only)

If you would like to mitigate the risk, you can always delete the API key after running this tool. This is obviously only an option when not running every 24hrs as the key will be useless.

Any security concerns/enquiries can be sent to alyssa@alyssaserver.co.uk

How does it work

I run a personal server 24/7 and it annoyed me that there weren't any easy ways to remove TikTok videos. I found Immich Tiktok Remover by Marcus-indrek Simmer and decided to turn it into a web app.

I also run a kubernetes cluster that's currently heavily under utilised. When you send an api requests to my server, it spins up a new deployment and runs the project on one of the nodes, therefore not loading my primary server too much. It won't be fast by any means, the nodes are low power mini PCs and can only do so much.
